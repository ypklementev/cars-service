# Car Listings Service

Тестовое задание: сервис сбора и поиска автомобильных объявлений.

Система собирает объявления с сайта, сохраняет их в базу данных,
предоставляет API и веб‑интерфейс для просмотра, а также Telegram‑бота с
поиском на естественном языке через LLM.

------------------------------------------------------------------------

# Архитектура

Проект состоит из нескольких сервисов:

### Backend (FastAPI)

Основной API‑сервис.

Функции: - хранение объявлений в базе - авторизация пользователей
(JWT) - защищённый API для получения списка автомобилей - upsert
объявлений (уникальность по `url`) - проверка подписи запросов от
воркера

Используемые технологии: - FastAPI - SQLAlchemy - PostgreSQL - Alembic
(миграции) - JWT авторизация

------------------------------------------------------------------------

### Scraper Worker

Отдельный сервис, который:

1.  периодически собирает объявления с **carsensor.net**
2.  переходит на страницу каждого автомобиля
3.  извлекает данные:
    -   марка
    -   модель
    -   год
    -   цена
    -   цвет
    -   ссылка
4.  нормализует данные
5.  отправляет их в backend API

Для безопасности запросы подписываются HMAC‑подписью.

------------------------------------------------------------------------

### Telegram Bot

Бот позволяет искать автомобили на естественном языке.

Пример запросов:

    найди красную BMW до 2 млн
    найди хонду
    покажи дешевые мазды

Логика работы:

1.  пользователь отправляет запрос
2.  LLM извлекает фильтры через **Function Calling**
3.  бот делает запрос в backend API
4.  бот возвращает найденные объявления

Используется: - aiogram - OpenAI‑совместимый API

------------------------------------------------------------------------

### Frontend

SPA‑интерфейс для просмотра объявлений.

Функции: - авторизация - таблица автомобилей - ссылки на оригинальные
объявления

Технологии: - React - Material UI - Axios

------------------------------------------------------------------------

### Database

PostgreSQL используется для хранения данных.

Миграции выполняются через **Alembic**.

При старте backend контейнера автоматически выполняется:

    alembic upgrade head

# Запуск проекта

## Требования

-   Docker
-   Docker Compose

------------------------------------------------------------------------

## 1. Клонировать репозиторий

    git clone <repo>
    cd cars-service

------------------------------------------------------------------------

## 2. Создать .env

Пример:

    DATABASE_URL=postgresql://postgres:postgres@db:5432/postgres

    JWT_SECRET=secret

    BOT_TOKEN=telegram_bot_token

    API_KEY=llm_api_key
    BASE_URL=https://api.artemox.com/v1

------------------------------------------------------------------------

## 3. Запустить сервисы

    docker compose up --build

------------------------------------------------------------------------

## 4. Доступные сервисы

Backend API:

    http://localhost:8000

Swagger:

    http://localhost:8000/docs

Frontend:

    http://localhost:5173

Telegram бот запускается автоматически.

# Архитектурные решения

### Docker‑ориентированная архитектура

Каждый компонент запускается как отдельный сервис:

-   backend
-   worker
-   bot
-   database
-   frontend

Это позволяет независимо масштабировать и обновлять сервисы.

------------------------------------------------------------------------

### Upsert объявлений

Объявления уникальны по `url`.

Если объявление уже существует, оно обновляется.

------------------------------------------------------------------------

### Подписанные запросы

Worker подписывает payload HMAC‑подписью.

Backend проверяет подпись перед записью в базу.

------------------------------------------------------------------------

### LLM Function Calling

Telegram‑бот использует LLM для извлечения фильтров из естественного
языка.

Это позволяет искать автомобили без строгих команд.

------------------------------------------------------------------------

### Alembic миграции

Миграции БД версионируются и автоматически применяются при запуске
backend контейнера.
